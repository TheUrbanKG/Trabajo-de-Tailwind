Base de Datos en Programación

¿A qué nos referimos cuando hablamos de base de datos?

Una base de datos es una recopilación de datos sistemática y almacenada electrónicamente. Puede contener cualquier tipo de datos, incluidos palabras, números, imágenes, vídeos y archivos. Puede usar un software denominado sistema de administración de bases de datos (DBMS) para almacenar, recuperar y editar datos.

Tipos de Base de datos

Las bases de datos son una parte esencial de cualquier sistema de información moderno ahora que todo gira entorno a la transformación digital y el Big Data. Desde aplicaciones de escritorio hasta aplicaciones web y móviles, todas ellas dependen de la capacidad de almacenar, acceder y manipular grandes cantidades de datos.

• Relacional: Utiliza tablas para organizar los datos en filas y columnas. Es ideal para datos estructurados y utiliza SQL para consultas.

• NoSQL: No sigue el esquema tabular tradicional. Es útil para grandes volúmenes de datos no estructurados y distribuidos.

• Centralizada: Todos los datos se almacenan en un solo lugar, lo que facilita la gestión pero puede ser un punto único de fallo1.

• Distribuida: Los datos se reparten en múltiples ubicaciones, mejorando la disponibilidad y redundancia1.

• Orientada a objetos: Los datos se representan como objetos, similar a la programación orientada a objetos

• Gráfica: Utiliza nodos y aristas para representar y almacenar datos, ideal para redes sociales y análisis de relaciones

Tipo de Gestores

• MySQL: Un SGBD de código abierto muy popular, especialmente en aplicaciones web.

• PostgreSQL: Un SGBD relacional y orientado a objetos conocido por su robustez y extensibilidad.

• Microsoft SQL Server: Un SGBD desarrollado por Microsoft, utilizado en aplicaciones empresariales.

• Oracle Database: Un SGBD muy potente y ampliamente utilizado en grandes corporaciones.

• MongoDB: Un SGBD NoSQL orientado a documentos, ideal para manejar grandes volúmenes de datos no estructurados.

Tabla, Registro y Campo

• Tabla: Es una colección de datos organizados en filas y columnas dentro de una base de datos. Cada tabla se enfoca en un tema específico, como “Clientes” o "Productos".

• Registro: Es una fila dentro de una tabla. Cada registro contiene datos relacionados entre sí.

• Campo: Es una columna dentro de una tabla. Cada campo contiene un tipo específico de información.

Modelo-Tablas y sus relaciones

Modelo de Base de Datos Relacional: Este modelo organiza los datos en tablas, que son colecciones de filas y columnas. Cada tabla representa una entidad, como “Clientes” o "Productos".

Tabla: Es una estructura que contiene datos organizados en filas y columnas. Cada fila (o registro) representa una instancia única de la entidad, y cada columna (o campo) representa un atributo de la entidad.

Relaciones entre Tablas: En un modelo relacional, las tablas pueden estar relacionadas entre sí mediante claves primarias y foráneas. Estas relaciones pueden ser de varios tipos:

○ Uno a Uno (1:1): Un registro en una tabla está relacionado con un solo registro en otra tabla3.

○ Uno a Muchos (1:N): Un registro en una tabla puede estar relacionado con múltiples registros en otra tabla3.

○ Muchos a Muchos (N:N): Múltiples registros en una tabla pueden estar relacionados con múltiples registros en otra tabla. Esto generalmente se maneja mediante una tabla intermedia3.
